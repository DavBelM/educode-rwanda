Default parameters

8 min read·Functions & Parameters

When you write a function, you decide what information it needs to do its job. Those pieces of information are called parameters. But what should happen when someone calls your function and forgets to pass one in?

Let's look at a greeting function. It expects a name, and builds a message from it.

greet.js
function greet(name) {
  return "Muraho, " + name;
}

greet(); // "Muraho, undefined"

Because no name was passed, name is undefined, and the message breaks. A default parameter gives the function something sensible to fall back on.

greet.js
function greet(name = "mwana") {
  return "Muraho, " + name;
}

greet();        // "Muraho, mwana"
greet("Aline"); // "Muraho, Aline"

The default only kicks in when the argument is missing or undefined. Pass a real value and it is used as normal.

Order matters. Parameters with defaults usually come last. If a default sits before a required parameter, you are forced to pass undefined to skip it — which defeats the purpose.

Defaults can be expressions

A default doesn't have to be a fixed value. It can be calculated — even from earlier parameters.

price.js
function total(price, tax = price * 0.18) {
  return price + tax;
}

total(1000); // 1180

Key points

  • A default parameter is a fallback used when no argument (or undefined) is passed.
  • Write it as function f(x = value) { }.
  • Defaults can be expressions, including ones built from earlier parameters.
  • Keep parameters with defaults at the end of the list.
